MybatisPlus.lambdaQuery.orderByDesc警告：Unchecked generics array creation for varargs parameter

这个警告 Unchecked generics array creation for varargs parameter（为可变参数创建了未检查的泛型数组）通常出现在 Java 使用 泛型 和 可变参数 (varargs) 结合的场景中。

在 MyBatis-Plus 中，orderByDesc 方法的签名通常是这样的：
orderByDesc(SFunction<T, ?>... columns)

当你传入 LearningLesson::getLatestLearnTime 时，Java 编译器会在后台隐式创建一个数组来存放这个参数。由于 SFunction 是一个泛型接口，Java 认为创建泛型数组是不安全的（Type Erasure 机制），因此抛出此警告。

这通常不会导致运行错误，代码是安全的。 你可以通过以下几种方式处理这个警告：

解决方法：使用 @SuppressWarnings 注解（推荐）

这是最简单且最常用的方法。既然你知道这里是安全的，可以直接告诉编译器忽略这个特定的警告。

你可以将其加在方法定义上，或者加在具体的变量声明上（如果适用）。

这是一个非常经典的 Java 面试题，也是理解 Java 泛型机制的核心。

简单来说，创建泛型数组是不安全的，根本原因在于 Java 的数组（Array）和泛型（Generics）的设计理念是冲突的。

具体冲突点如下：

  1. 数组是协变的（Covariant）且具体化的（Reified）。
  2. 泛型是不变的（Invariant）且会被擦除的（Erased）。

1. 什么是“泛型擦除”？

Java 的泛型是伪泛型。在编译期间，编译器会检查类型安全，但在运行期间（JVM 中），所有的泛型信息都会被擦除。
